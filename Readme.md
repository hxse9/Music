# 🇰🇷 한국 음악 추천 및 공유 커뮤니티 플랫폼

본 프로젝트는 한국 음악(K-Pop, 인디, 힙합, 발라드, 트로트 등)을 사랑하는 사람들이 모여 자신의 "최애곡"을 소개하고 추천을 공유하는 웹사이트입니다. 직접 추천사를 작성하고 다른 사람의 음악 취향을 엿볼 수 있는 소셜 음악 공유 플랫폼을 목표로 합니다.

---

## 🚀 기획 의도 및 핵심 기능 (Features)

이 웹 플랫폼은 단순한 게시판을 넘어 '음악 추천'이라는 테마에 집중한 커뮤니티 기능을 제공합니다.

### 1. 🎵 음악 추천 피드 (Music Feed)
*   **음악 포스팅 기능:** 곡명, 아티스트명, 장르 태그(예: `#댄스`, `#감성`, `#새벽에듣기좋은`), 추천 이유, 그리고 유튜브 링크를 입력 및 포스팅할 수 있습니다.
*   **미디어 재생:** 유튜브 영상 URL 등을 통해 게시글 내에서 바로 음악(또는 뮤직비디오)을 들어볼 수 있습니다.
*   **앨범 커버 (Optional):** 해당 곡의 앨범 자켓 이미지 또는 뮤직비디오 썸네일을 피드 형태로 예쁘게 제공합니다.

### 2. 👥 커뮤니티 상호작용 (Social Features)
*   **좋아요(Like) 및 댓글(Comment):** 마음에 드는 음악 추천 게시글에 좋아요 피드백을 남기고, 댓글을 통해 의견을 나눌 수 있습니다.
*   **내 플레이리스트 (My Page):** 내가 좋아요한 추천 음악들을 모아 볼 수 있는 보관함 기능을 제공합니다.

### 3. 🔍 음악 검색 및 필터 (Search & Filter)
*   아티스트별, 곡명별 검색뿐만 아니라 지정된 **해시태그**를 기반으로 분위기나 장르에 맞는 곡을 탐색할 수 있습니다.

### 4. 👤 사용자 인증 및 관리 (Auth)
*   개인별 계정 생성(회원가입, 로그인, 로그아웃) 시스템.
*   비회원도 피드 열람은 가능하지만, 추천(글 작성) 및 좋아요/댓글 기능은 회원에게만 허용합니다.

## 🛠 작업 및 개발 방법 (Tech Stack & Work Flow)

본 프로젝트는 가볍고 빠르게 웹 플랫폼의 모든 뼈대를 세울 수 있는 **Python 기반 서버와 SQLite 로컬 데이터베이스 환경**을 채택합니다.

### 사용 기술
*   **Backend:** Python 3.x, Flask (프레임워크)
*   **Database:** SQLite3 (가볍고 별도의 서버 구축 없이 파일 기반으로 동작)
*   **Frontend:** HTML5, CSS3 (모던 UI/UX 적용), Vanilla JavaScript

### 작업 단계 가이드라인

1.  **DB 스키마 구축 단계 (현재 완료됨)**
    *   `music.db` 데이터베이스 및 스키마(`schema.sql`) 설계. 사용자와 게시글, 댓글 등을 포함하는 논리 구조를 구성합니다.
2.  **백엔드 서버 세팅 시작**
    *   Python 가상환경(venv) 생성 및 `pip install flask` 실행.
    *   `app.py` 라우터를 작성하고 데이터베이스 파일과 연결(DB Connection)합니다.
3.  **UI/UX 디자인 및 템플릿(HTML) 구성**
    *   초기 메인화면(Home), 포스트 작성 화면(Form), 회원가입/로그인(Auth) 디자인을 구현합니다. CSS를 이용해 음악 앱처럼 세련된 Dark Mode 또는 깔끔하고 청량한 컬러를 사용합니다.
4.  **백엔드 프론트엔드 연동**
    *   사용자가 HTML 폼에서 글을 작성하면 Python Flask가 이를 받아 DB에 `INSERT` 합니다.
    *   메인화면 접속 시 DB에서 작성된 데이터를 `SELECT` 하여 목록(Feed)으로 뿌려줍니다.
5.  **테스트 및 보완**
    *   각 기능별(로그인 세션 유지, 댓글 작성 등) 에러를 처리하고 수정하는 등 안정성을 확보합니다.

## 🗄 데이터베이스 구조 (Database Schema)

음악 자료 및 커뮤니티 상호작용 저장을 위한 데이터베이스(`music.db`)의 기본 구조는 다음과 같습니다. (자세한 내용은 `schema.sql` 참고)

*   `users` : 플랫폼 회원 정보 (ID, 이메일, 닉네임, 비밀번호 등)
*   `posts` : 음악 추천 작성글 본문 (곡 정보, 추천 이유, 유튜브 링크 등)
*   `comments` : 타 회원이 작성한 댓글 내역
